# 個人檔案產生器 — 給同事的使用說明

這是一套讓 AI agent（Claude Code 或 Hermes agent 都可以）幫你把「個人信息登記表」
做成好看的 HTML 個人檔案、或是補一份格式一致的 xlsx 的工具。用途：

- 手上已經有一份 xlsx 格式的個人信息表，想轉成方便閱讀分享的網頁版。
- 只有一個人名（例如某位訪客、某位教授），想請 agent 上網查資料，
  幫你整理成跟公司既有格式一致的「個人信息表」（同時給 HTML 網頁版跟 xlsx 版）。

這份「使用說明」是給人看的；同一個資料夾裡還有一份 `SKILL.md`，
是寫給 agent 看的技術指令，兩份搭配使用，你不需要去讀 `SKILL.md`。

---

## 這是什麼、為什麼這樣做

以前每次要做一份人物背景資料，都要重新排版、重新想樣式，容易每份長得不一樣，
也容易漏掉「這筆資料到底是哪裡查到的」這種重要但麻煩的紀錄工作。

這套工具把「資料」跟「排版」分開：agent 先把查到的事實整理成一份簡單的
`profile.json`（姓名、學經歷、現任職位…等欄位），再用固定的版型自動產生
HTML 網頁跟 xlsx 檔案。好處：

- 兩份輸出（HTML／xlsx）保證資料一致，不會改了一邊忘記改另一邊。
- 每次產生的網頁／表格版型都一樣，不會這次一個樣子、下次又長得不一樣。
- 欄位固定為 10 項，查無資料的欄位一律標「-」，不會被誤以為是猜測出來的真實資料。
- 每一份檔案底部都會列出資料來源網址，方便日後查證。

這套方法不綁定特定產品——它是照著 [agentskills.io](https://agentskills.io)
這個開放格式寫的，Claude Code、Hermes agent，或其他支援同一種格式的 agent
都能用同一份 `visitor-profile-builder/` 資料夾。差別只在「這個資料夾要放在哪裡」，見下方。

## 開始使用前（每個人只需要做一次）

### 第一步：把 `visitor-profile-builder/` 資料夾放到你的 agent 找得到的地方

- **用 Claude Code 的人**：複製（或 symlink）整個資料夾到你專案底下的
  `.claude/skills/visitor-profile-builder/`。
- **用 Hermes agent 的人**：複製整個資料夾到
  `~/.hermes/skills/visitor-profile-builder/`。
- **其他支援 agentskills.io 格式的 agent**：查該 agent 文件裡的 skill 目錄慣例。

### 第二步：確認 Python 環境有需要的套件

跟你的 agent 說一聲，或自己在終端機跑：

```bash
python3 -m pip install openpyxl jsonschema Pillow
```

（刻意寫成 `python3 -m pip` 而不是 `pip`：很多人電腦上的 `pip` 跟 `python3`
指向不同的 Python——例如一個是 miniconda、一個是 Homebrew——那會裝進一個
執行程式時根本不會用到的環境，症狀是「裝好了卻說找不到套件」。）

（`Pillow` 只有在要把照片嵌進 xlsx 檔時才需要，沒有照片可以省略；
`jsonschema` 用來檢查資料格式，建議裝，缺了會跳過部分檢查。）

如果你的電腦跳出「externally-managed-environment」之類的錯誤（macOS 用
Homebrew 裝的 Python 很常見），不要硬裝，改建一個一次性的虛擬環境就好，
不影響系統 Python：

```bash
python3 -m venv /tmp/vpb-venv && /tmp/vpb-venv/bin/pip install openpyxl jsonschema Pillow
```

之後請 agent 用 `/tmp/vpb-venv/bin/python3` 來執行 `scripts/` 裡的程式即可。

### 第三步（選用）：只有需要 PNG 圖檔才要做

這步需要電腦上有 **Node.js**（HTML 跟 xlsx 都不需要，只有截圖用得到）。
沒裝過的話先去 [nodejs.org](https://nodejs.org) 裝一份，或用 `brew install node`；
已經有的話 `node -v` 會印出版本號。

然後在 `visitor-profile-builder/scripts/` 資料夾裡跑一次（只需一次）：

```bash
npm install
```

這會裝好 `puppeteer-core`。截圖程式會**先試著用你電腦上已經裝好的 Chrome**，
所以大部分情況下你不需要另外裝瀏覽器。只有在真的沒有 Chrome 時（例如在沒有
桌面環境的 Linux 主機上），才需要多跑下面這行，讓它自己下載一份專用的
Chrome（不需要管理員權限，會放在 `~/.cache/puppeteer/`）：

```bash
npx puppeteer browsers install chrome
```

#### 部署在雲端 Linux／Ubuntu 純 CLI 環境

沒有桌面環境的機器要多做一件事，而且**是必要的、不是選用的**：

```bash
sudo apt install -y fonts-noto-cjk        # 中文字型
npx puppeteer browsers install chrome     # 這種機器通常沒有系統 Chrome
```

**中文字型一定要裝。**乾淨的 Ubuntu server 映像檔完全不含中文字型，
Chrome 會退回 DejaVu Sans——那套字型沒有任何中文字，於是 PNG 裡的每一個
中文字都會變成一個空心方框（□□□）。麻煩的是這個過程不會報任何錯，
檔案照樣產出，只是整張圖不能看。

HTML 跟 xlsx 不受影響（它們是在收檔案的人電腦上顯示的，用的是那台的字型），
**只有 PNG 會把字型烤進圖片裡**，所以只有 PNG 有這個問題。

Docker／以 root 執行也可以，截圖程式在 Linux 上會自動加上必要的
`--no-sandbox` 參數。產完 PNG 後請 agent 直接把圖顯示出來看一眼再收工——
截圖出錯時通常不會報錯，只能用看的。

**懶得一項一項確認的話**，skill 內附一支腳本會把上面全部做完並自我檢查：

```bash
cd <skill 目錄>
./scripts/verify-on-ubuntu.sh --dry-run   # 先看它會裝什麼、動到什麼
./scripts/verify-on-ubuntu.sh             # 實跑
```

它會在裝字型「之前」和「之後」各截一張 PNG 放進 `~/vpb-verify-out/`，
讓你直接比對。第一張的中文預期就是空心方框——那是對照組，不是壞掉。

## 日常怎麼用

### 情境一：我手上已經有一份 xlsx 個人信息表，想轉成網頁

直接把檔案路徑給 agent，跟它說：

> 把這份 xlsx 轉成 HTML 個人檔案

agent 會讀出裡面的資料，套用固定版型，產生一份網頁檔給你。

### 情境二：我只有一個人名，想請 agent 幫我查資料建檔

跟 agent 說類似這樣的話：

> 請網路搜尋，建立 XX大學 XX系 XX教授，並建立個人檔案

agent 會：
1. 上網搜尋確認這個人的身分（避免同名同姓查錯人）
2. 整理學經歷、現任職位等資料，查不到的欄位會誠實標「-」
3. **順便找照片**——這是預設動作，不用特別開口要求。agent 會上網找候選照片，
   **先讓你確認要用哪一張（或兩張都用）**，才會正式嵌入檔案；如果都找不到
   有把握是本人的照片，就會維持無照片版本。你也可以直接說「不用附照片」跳過這步。
4. 產生 HTML 網頁版個人檔案，並列出所有資料來源網址
5. 如果你想要同時有 xlsx 版本，直接跟它說「也幫我做一份 xlsx 版本」即可

### 情境三：想要 PNG 圖檔（例如要貼到聊天室或簡報）

跟 agent 說「幫我轉成 PNG」或「除了 html、xlsx，我還要 png」之類的話即可。
agent 會把 HTML 網頁截圖成一張完整的 PNG 圖片，跟 HTML 長得一模一樣，
放在 HTML 檔案旁邊、同一個檔名。

## 你會得到什麼樣的檔案

- **HTML 個人檔案**：卡片式版面，姓名／現任職位／基本資料徽章在最上方，
  接著是教育經歷、主要履歷等表格區塊，最下面有備註跟資料來源清單。
  可以直接用瀏覽器打開，也可以轉寄給同事看。
- **xlsx 個人信息表**：版面對齊公司原本用的「個人信息登記表」範本
  （合併儲存格、姓名／性別等欄位位置都一致），方便延續舊有的歸檔習慣。
- **PNG 圖檔**（選用）：**手機閱讀版型**——直式、字體較大，兩個表格會改成
  一筆一塊的條列（例如每段學歷自成一小段，院校粗體、專業／學歷／學位列在
  下方），在聊天室裡直接看就清楚，不用放大。方便貼到聊天室或簡報。

三種檔案的樣式（顏色、字體、欄位標籤）全部來自 `scripts/` 裡固定的版型邏輯，
不管是誰、用哪個 agent 產生的，長相都會保持一致。

**想先看看長什麼樣子**：用瀏覽器打開 `assets/profile.example.html`。那是一份
用虛構人物「王小明」產生的完整範例，10 個欄位都在（包含查無資料而顯示「-」
的那幾格），跟你之後拿到的檔案版型完全一樣。

## 常見狀況

- **agent 說「驗證失敗，未產生檔案」**：這是刻意的保護。檔案有固定的欄位
  規格（例如教育經歷最多 5 列、現任職位最多 3 個、主要履歷最多 10 筆、
  照片最多 2 張、資料來源至少 1 筆），
  違反時會擋下來而不是產出一份版面壞掉的檔案。直接請 agent 依訊息修正即可。
- **看到 ⚠️「jsonschema 未安裝，已跳過 schema 結構驗證」**：檔案還是會照常
  產出，但只跑了內建的基本檢查，欄位數量上限、型別這些沒有被驗到。
  照上面第二步把 `jsonschema` 裝起來就會消失。這是實務上最常見的一則警告，
  因為 macOS 內建的 python3 通常沒有它。
- **有些欄位寫「-」**：代表 agent 查不到這項資料。這是刻意的設計——寧可
  誠實留空，也不會用猜的填進去。個人檔案固定就是這 10 個欄位（姓名、性別、
  出生年月、生肖、籍貫、聯繫方式、教育經歷、現任職位、主要履歷、照片），
  查無資料的欄位不會被省略，一律顯示「-」。
- **備註欄提到「依新聞報導推估」**：代表這項資訊不是官方公告，
  是從新聞或公開演講內容合理推斷出來的，需要更精確資訊建議自行查證。
- **照片畫質很差**：非官方來源的照片（例如系所網頁上很舊的小圖）
  解析度通常都不高，這是正常現象，備註欄會註明照片並非官方授權提供。
- **想幫不同專案調整資料夾名稱／版型顏色**：直接跟你的 agent 說即可，
  它可以直接修改 `scripts/` 裡的版型設定，之後產生的檔案都會套用新樣式。
- **換了一個新的 agent 產品**：只要它支援 agentskills.io 這個開放格式，
  把 `visitor-profile-builder/` 資料夾放到它的 skill 目錄下就能用，
  不用重寫任何東西。

---

有任何問題，或想調整版面／欄位，直接跟你的 agent 說即可。
